Go, Go, Go The Core

November 28, 2013

Due to me recent evolutions I’m refocusing my development of my core. Instead of focusing on PHP and Javascript, I’m shifting to Go. While Javascript and PHP are necessary for immediate work, Go is a long term investment that will pay much larger dividends in the end. The working knowledge I have of PHP and Javascript will more than sustain me for the next few months.

The First Few Steps

After a suggestion from a fellow Drupaler to dive head first into Go, I have a renewed energy for learning the language and incorporating it into my work. I’ve had to do several systems tasks over the past few months and I’ve been using PHP to accomplish them. My mentality then was that I was already investing so much in this language, why not use it for as many things as possible. The problem with that is the resources for accomplishing systems tasks with PHP isn’t what PHP is commonly used for which means there isn’t a lot of community support for that. Go is a completely different beast. I’ve only done a part of the Go tour and I can already tell this language will be able to tackle the problems I’ve been trying to solve with PHP in a much more elegant way.

What About PHP?

I’m not abandoning PHP. I obviously will still use it at work, but I still see value in using it for other projects. Not everyone I’ll work with will know Go, which is why PHP is still in my core. With my new focus on being full stack, using PHP as my primary language isn’t going to cut it.

The same goes for Drupal. Every toolbox needs a good set of tools that can handle a variety of tasks at different levels. Some tools are for sketching out ideas and others are for getting a large amount of work done quickly. PHP feels more like the sketching tool and Go feels like the robust tool to get a large amount of work done. Building prototypes in PHP and Drupal is fast. It would also assist in shifting my mentality from trying to do things ‘The right way’ in Drupal. As I’ve said, there isn’t ever a ‘right’ way to do something, just a more correct way.

Learning Go

From the Go tour so far I am confident that I can learn Go without the use of books, sticking strictly to the internet resources available. My knowledge level of general programming principles is far different than it was a few months ago so the basic introductory texts are pretty dry at this point. Tell me the basics and let’s dive into some complex stuff.

That said, there is an interesting resource I did come across in my search for learning material, an ebook that an internet book. For the most part I’ve considered books things that one would purchase from a published, not so much self published books, but how different are these things? In reality though, I feel that the Go tour covers most if not all of what the book goes over.

Either way I’m excited to finally get a deeper dive into this language. I’ve been itching to use something a bit more robust than PHP for a while so it’s definitely going to be a great experience.
