Unique Tools Workflow & Development

November 14, 2013
By: skriptble

I love the development environment I use. When I talk to people about it, several don’t understand the choices I made in the tools I’ve selected. The one tool that really baffles some is my use of VIM over an IDE. I’ve tried IDE’s before. I’ve used Eclipse, Aptana, Sublime, TextWrangler, none of them feel like a right fit. That’s just the first tool, from my editor to my shell, I like to ride a bit outside the norm.

It starts with your workflow

The reasons I use my unique set of tools stems from the way I develop applications. At work, I build my applications on a development server or a VirtualBox style set up. Personal work I usually do somewhere between VirtualBox and locally installed applications.

VIM

The most important tool in a developer’s toolbox is a text editor. It’s hard to write code if you don’t have a thing to write code in. For this task I picked VIM. I could have easily wound up with emacs, but through some series of events I fell into VIM. So why not use an IDE? Something full featured with nice things like syntax highlighting and autocomplete? Well, I do actually. My VIM environment is configured with a bunch of plugins that give me the features I want without the ones I don’t. The main feature I dislike about IDE’s is FTP. I hate FTPing files around. Vagrant can help mitigate this problem, and it works great when you’re working alone. When you’re on a team, though, having to do a git commit and push or FTP the files and run an update script is time consuming. Sometimes you want to copy and paste the link in your browser to a coworker and ask them to take a look.

That said, I’m sure some of the newer IDEs have solutions to these problems. That’s great an awesome, and maybe I’ll give them a try sometime. For now, VIM is my editor of choice. It has the nice bonus of being able to edit almost any text file I come across too. Score!

ZSH

The second most important tool to me is the shell. Since I work on servers with no GUI, most of my day is spent in the shell, either inside of VIM or running various command . A shell that gives me the ability to work efficiently pays dividends. I found ZSH a number of months ago and I haven’t looked back. I’m still discovering new features of ZSH, some of which are present in BASH, and some that are just simpler to implement in ZSH. Since I use VIM, I’ve decided to switch the default emacs bindings to VIM bindings, which will hopefully increase my productivity in the end.
