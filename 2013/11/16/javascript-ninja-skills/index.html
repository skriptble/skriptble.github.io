Javascript Ninja Skills The Core Languages

November 16, 2013

I struck lucky this week and received another 20% off coupon from Barnes and Noble. I am at the point where focusing on my core is important, so I’m trying to buy books that help me focus on that core. There is a Go book I’ve been eyeing for a while, but I’m determined to learn Go and most of Erlang using the resources of the web, not books. Multiple avenues of learning is going to be very important on this journey. That left me with a problem: Should I pass up a chance to save 30% on a book?

Skill Level: Ninja

Of course I didn’t pass up saving 30% off a book. I have a habit of identifying books and thinking about them overtime. Now that I get newsletters from multiple publishers, I have some idea of the books I want before I get to the store. This week’s choice was a Javascript book co-authored by the creator of jQuery, Secrets of the Javascript Ninja. Javascript is one of the languages in my core, so I’m not deviating or cheating like I did last week.

Admittedly, I did pick up a couple of books on regular expressions. After some evaluation I decided that it made more sense for me to sharpen up my Javascript. As a bonus the book I bought has a chapter dedicated to regular expressions. I see regular expressions as one of those auxiliary technologies I need to learn. Knowing six programming languages isn’t much use without the addition of other technologies.

I’m excited to have this book though. While web applications powered entirely by the server are cool and all, having some sharp Javascript skills definitely makes a difference. As with most things, it’s not one or the other. Building great web applications requires both server-side and client-side code. Understanding the lower level languages is also important. Dart 1.0 was released a couple of days ago and I read up on some of the chatter about it’s release. I tried it out about a year ago, but I didn’t end up getting involved in it. I wasn’t interested in front end development at the time.

Overtime Dart might gain more traction over Javascript, but I still see value in learning vanilla Javascript. There’s also a chance it will help me learn Dart as well. I stuck with PHP instead of switching to python, ruby, C#, or Java. I’ve invested quite a bit of time in learning Javascript already as well. If you’re just starting out, perhaps it wouldn’t be a bad idea to go grab the Dart download and try out the tutorial. Especially if you want to get to building things right away.
