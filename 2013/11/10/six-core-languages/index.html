Six Core Languages

November 10, 2013

I’m a goal driven individual. Without goals I often find myself drifting into strange places that are usually unproductive and lead to a state of complacency. Setting insane goals is an awesome way to push myself to the edge, so I’ve decided to set one. I want to learn 6 core languages.

On its face that doesn’t seem too hard. There are plenty of people out there who become a polygot. It makes sense and most people probably should be a polygot. Having more than one tool in your toolbag makes everything you build that much better. Sure you can build things with only screws, but having a hammer and a saw in there will make it easier.

The Core

The reason my goal is a touch insane is that I don’t want to learn closely related languages. The languages I picked are a little all over the place. So here’s the list: PHP, Javascript, Go, C, C++, Erlang. I’m cheating a little because I already know a hefty amount of PHP and Javascript and I’m a novice C and C++ guy. Other popular languages are missing like Java, Python, and Ruby. I’m not saying I won’t learn those, but I don’t intend on having them be part of my core.

The strangest one in this mix is definitely Go. It’s a modern language compared to most others, it was built by some pretty brilliant guys, and it fascinates me. I find python and ruby interesting, but Go has a different feel to it.

Second strangest is probably Erlang. When I decided I wanted to diversify my language toolbox, I realized I should probably diversify the types of languages as well. Erlang is a popular functional language, and I think it makes a useful addition to my core.

C++ is the one language I’m not sure of, but I feel as if it goes along with C. I might swap it out for a different language seeing as I have C for the low level stuff and Go for the slightly higher level stuff (unless I’m misunderstanding Go).

As I stated before this is my core, the languages I want to be able to wield at a moments notice and be fluent in. I envision Python and Ruby, maybe even a classic language like Smalltalk will be in my second tier of languages. Those which I can read and write, but not at the same level as my core, although they will most likely eventually move into the core as far as proficiency is concerned. On the outer tiers I see languages like Java and C#. I’ll be able to read and perhaps someday develop in them, but I don’t see them in my core.

While I do see the value in diversifying my language toolbox, I don’t see the need to accessorize my toolbag. That’s read, Text Editor and IDE. I’m a VIM guy. I like writing things in VIM, I’m writing this blog post in VIM, and I’ve got the IDE features I want set up in VIM. Any features I’m missing I can always add later. EMACS is cool, and some of my friends are EMACS people, I’m not. It’s a similar situation with platforms. I’m not saying that I refuse to learn Windows or work on a Windows platform, but I’m going to focus most of my energy on linux and unix platforms.

DevOps

My half of my core are languages known as “system languages”. Modern developers don’t get to sit in a box, build an application, and rely on a sysadmin to deploy and scale it for them. We have the cloud now, which means the applications we write can scale themselves and with a little help from continuous integration, deploy themselves. The barriers of what languages are used for what tasks are starting to melt. I’ve seen too many stacks falling over or different parts of stacks falling over to not recognize the importance of cloud applications.

There should be no buttons pushed by people, traffic logs or analytics analyzed by hand, or code deployed manually. It isn’t practical nor efficient. I’m bad at doing repetitive things, machines aren’t. Figuring out when an application might get heavier traffic? That sounds like a lot of analysis of several different angles. That sounds like a math problem.

All in all, I think I’ve got a solid core, but that’s for right now. I could wake up tomorrow morning and realize I need to swap C++ with Python. I have little desire to be idealistic about my core. This early in the game there is quite a heft of flexibility. Might as well use it.
