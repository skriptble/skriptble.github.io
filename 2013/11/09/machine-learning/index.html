Automation & Machine Learning

I enjoy automating processes. I recognize that I am not efficient at doing repetitive tasks; I am efficient at automating repetitive tasks. Even if things only need to be done once or twice, automating it forces a different way to think about the task at hand. It requires code and data that a machine can read. Property names are standardized instead of using in the moment random names. Changing these naming schemas requires more before-thought, so they need to be designed with flexibility in mind. Having this flexibility helps the system as a whole.

Blend some crafty regular expressions with some code generating templates, sprinkle with a few well crafted shell scripts, or use a single application for making large batches. Bake for an hour and serve automation while warm. No need for external documentation, that stuff gets moldy real quick.

I’m currently reworking an internal system at work used for people profiles. Instead of simply rebuilding what was already there I decided to do some refactoring as I went along. Knowing that this suite of modules would be used all over the place I decided to make it easy for our Drupal framework to be able to handle most of the tasks itself. Eventually, we want to have an ecosystem of applications that push updates to profiles out too all the sites that have that profile in real time. The sites themselves would know how to handle or retrieve the data, enabling us to do manual updates if need be.

I haven’t gotten too deep into this profile system before this project. The most work I had done was to grab the data being spit out and process it through Drupal, ensuring it was output and formatted correctly. The more I view the data we’re dealing with and hear about different enhancements that are requested, the more I realize that machine learning will be handy now and essential in the future.

Aside from that I see other uses for machine learning, such as analyzing our logs to determine when trouble might be heading our way, or determining when the application needs to scale and which layers need to scale.

I could have a very wrong interpretation of what can be done with machine learning but I think these tasks can be accomplished with it. By automating all these things I can get back to focusing on building awesome applications. I consider this a valuable skill to have in my toolbox, but part of me wonders if I’ll be able to fit another language in there.

Categories aside, there is definitely overlap in concepts between the languages. Anyway, I wasn’t planning on shoving Python in there. That said, machine learning is definitely an important skill to have. This will require more thought.

In the end I know I will use machine learning here and there and the likelihood of encountering Python in the wild is high. I will definitely be posting about those 6 core languages too.
